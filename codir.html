Что такое кодировка?

В компьютерах всё хранится в виде чисел. Кодировка - это способ представления символов (букв, цифр, знаков препинания и т.д.) в виде этих чисел. Другими словами, это таблица соответствия между символом и его числовым кодом.

Без кодировок компьютеры не смогли бы отображать и обрабатывать текст. Правильная кодировка необходима для корректного отображения текста на разных устройствах и платформах.

Исторические кодировки

В ранних компьютерах использовались разные кодировки, часто специфичные для конкретного языка или страны. Это приводило к проблемам совместимости.

  • ASCII (American Standard Code for Information Interchange): Одна из самых старых и распространенных кодировок. Изначально содержала 128 символов (буквы латинского алфавита, цифры, знаки препинания, управляющие символы). Использует 7 бит на символ.
  • КОИ-8 (Код обмена информацией, 8-битный): Семейство 8-битных кодировок, разработанных для представления кириллических алфавитов (русского, болгарского и т.д.). Существуют различные варианты, например, КОИ-8р (для русского).
  • Windows-1251: Еще одна 8-битная кодировка для кириллицы, разработанная Microsoft. Широко использовалась в Windows.

Проблема этих кодировок заключалась в том, что они не могли одновременно представлять символы разных языков. Для каждого языка или группы языков приходилось использовать свою кодировку.

Unicode: Универсальная кодировка

Unicode - это стандарт, предназначенный для представления символов практически всех письменных языков мира. Он присваивает каждому символу уникальный числовой код (code point).

Unicode определяет только соответствие между символами и кодовыми точками. Для представления Unicode в компьютерах используются разные кодировки:

  • UTF-8 (Unicode Transformation Format, 8-bit): Самая распространенная кодировка Unicode в интернете. Использует переменное количество байт на символ (от 1 до 4). Совместима с ASCII для английских символов.
  • UTF-16 (Unicode Transformation Format, 16-bit): Использует 2 или 4 байта на символ. Широко используется в Windows и Java.
  • UTF-32 (Unicode Transformation Format, 32-bit): Использует 4 байта на символ. Проще в обработке, но занимает больше места.

UTF-8 является предпочтительной кодировкой для веб-страниц, так как она обеспечивает хорошую совместимость, эффективно использует место для английского текста и поддерживает все символы Unicode.

Практическое применение

Выбор правильной кодировки важен для:

  • Веб-разработки: Укажите кодировку UTF-8 в HTML-документе ( <meta charset="UTF-8">) и сохраняйте файлы в этой кодировке.
  • Баз данных: Выбирайте кодировку Unicode (например, UTF-8) для хранения текстовых данных.
  • Обработки текста: При чтении и записи текстовых файлов указывайте правильную кодировку.

Неправильная кодировка может привести к отображению "кракозябр" (нечитаемых символов).
